# CODEX.md

## プロジェクトの目的

このプロジェクトは、家庭で毎日使える「新しい家族のようなチャットボット」を作るためのものです。
ただ返答するだけではなく、短く自然に話し、相談、雑談、予定整理、気分転換、調べものを手伝う存在を目指します。

## 今回の再構築で大切にしたこと

- 1つの巨大な `chatbot.py` から、機能ごとの小さなファイルへ分割した。
- ローカルLLM前提の設定を取り除き、OpenAI APIを中心にした構成へ整理した。
- 家庭で音声利用しやすいよう、返答は短めで自然な日本語になるようにした。
- 外部APIの待機処理にタイムアウトを入れ、止まりっぱなしになりにくくした。
- 会話履歴を保存しつつ、長くなったら要約して次回以降に引き継ぐ流れを残した。
- 会話ログや `.env` はGitに入らないよう `.gitignore` を追加した。
- APIキーを毎回設定しなくてよいよう、`.env` の自動読み込みに対応した。

## 現在の構成

- `chatbot.py`
  - 入口。`family_chatbot.app.main()` を呼び出すだけにしている。
- `family_chatbot/app.py`
  - 入力、応答生成、読み上げ、音声待機の全体制御。
- `family_chatbot/config.py`
  - 環境変数、既定モデル、呼びかけフレーズ、家庭向けシステムプロンプト。
- `family_chatbot/llm.py`
  - OpenAI APIを使った返答生成。検索や「続けて」の処理もここで扱う。
- `family_chatbot/memory.py`
  - 会話履歴の読み書きと要約。
- `family_chatbot/speech.py`
  - SpeechRecognitionを使った音声入力。単発入力と呼びかけ待機に対応。
- `family_chatbot/tts.py`
  - tts.quest の VOICEVOX APIで音声を生成する。
- `family_chatbot/audio.py`
  - ビープ音と音声ファイル再生。
- `family_chatbot/search.py`
  - SerpAPIによるWeb検索。

## 設計メモ

家庭用なので、機能の多さよりも「話しかけやすさ」を優先する。
返答は音声再生される前提で、長すぎないことが大切。
会話相手としての人格は、親しみやすいが踏み込みすぎない距離感にする。

外部APIに依存するため、障害時のふるまいも重要。
TTSや検索が失敗しても、アプリ全体が落ちないように各モジュールで例外を受け止める。

## 今後の改善候補

- `.env` 読み込み対応を追加する。
- 家族メンバーごとの名前、好み、予定を保存するプロフィール機能を作る。
- 「おはよう」「ただいま」などの生活シーン別の返答を強化する。
- 音声待機中のスレッド停止をより確実にする。
- TTS APIを差し替えやすいように抽象化する。
- Raspberry Pi向けのセットアップ手順を詳しく書く。
- テストを追加し、APIなしでも会話履歴や検索判定を確認できるようにする。

## 開発時の注意

- `conversation_log.json` には個人的な会話が入るのでGitに含めない。
- APIキーはコードに直接書かず、`.env` または環境変数で渡す。
- 音声やマイク周りは環境差が大きいため、失敗してもテキスト会話が使える状態を保つ。
- 返答スタイルを変える場合は `family_chatbot/config.py` の `DEFAULT_SYSTEM_PROMPT` を確認する。
